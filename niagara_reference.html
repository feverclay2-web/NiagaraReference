Niagara 系统架构总览

Niagara 是 UE4.20+ / UE5 引入的新一代粒子系统，替代旧版 Cascade。它采用模块化堆栈架构，所有行为由可插拔的模块从上到下依次执行。理解这个架构是掌握 Niagara 的第一步。

▾ Niagara 四层嵌套结构（点击展开详情）
📦 Niagara System（系统）
最外层容器

System 是可以直接放入关卡的资产。一个 System 包含一个或多个 Emitter。

  • System Spawn — 系统创建时执行一次
  • System Update — 每帧执行，控制全局参数
  • User Parameters — 暴露给蓝图/Sequencer 的参数
  • 一个 System 内的 Emitter 共享时间线和空间
▼
🎯 Emitter（发射器）
中间层

每个 Emitter 定义一种粒子的行为。一个 System 可以包含多个 Emitter 来组合复杂效果。

  • Emitter Spawn — 发射器创建时初始化
  • Emitter Update — 每帧更新发射器状态
  • Execution State — Active / Inactive / InactiveClear / Complete
  • 可设置 Local Space 或 World Space
  • 每个 Emitter 独立管理自己的粒子池
▼
✨ Particle（粒子）
最内层

每个粒子是 Emitter 产生的一个独立实体，拥有自己的属性集。

  • Particle Spawn — 粒子出生时执行一次（位置/速度/颜色/大小等初始值）
  • Particle Update — 每帧对每个存活粒子执行（重力/阻力/颜色渐变等）
  • 核心属性：Position, Velocity, Color, Size, Age, Lifetime, NormalizedAge…
  • 可自定义属性（右键 → Add Parameter）
▼
🖼️ Renderer（渲染器）
输出层

Renderer 决定粒子的可视化方式，读取粒子属性并渲染到屏幕。

  • Sprite Renderer — 2D 面片/Billboard（最常用）
  • Mesh Renderer — 3D 模型实例化
  • Ribbon Renderer — 拖尾/带状（刀光、轨迹）
  • Light Renderer — 每粒子一盏动态灯
  • Component Renderer — 任意 UE 组件
  • Decal Renderer — 贴花投射
💡
核心理解：Niagara 的一切行为都是 Module（模块）。模块按 从上到下的顺序在对应的组（Spawn/Update/Render）中执行。后面的模块可以读取并覆盖前面模块写入的参数值。这就是「堆栈」的含义。
🔄
粒子生命周期

每个粒子从出生到死亡，经历一系列明确的阶段。理解这个流程，就能准确判断「我的参数应该放在哪个组里」。

▾ 粒子完整生命流程
🔵
System Init
系统创建，初始化全局变量
🟢
Emitter Spawn
发射器启动，设置生成速率
🟡
Particle Spawn
粒子出生，初始化属性
🟠

每帧更新位置/颜色/大小
🩷
Render

🔴

⏱️
关键时间变量：
Particles.Age — 粒子已存活的秒数
Particles.Lifetime — 粒子总寿命（秒）
Particles.NormalizedAge — Age / Lifetime，范围 0→1，用于曲线采样
当 NormalizedAge ≥ 1 时粒子被销毁（除非手动覆盖 Kill 逻辑）

Emitter 执行状态

Active
正常模拟 + 可以生成新粒子
Inactive
继续模拟现有粒子，但不再生成新粒子
InactiveClear
销毁所有现有粒子，然后进入 Inactive
Complete
完全停止模拟和渲染

模块堆栈详解

Niagara 的核心思想是堆栈模型：每个组（Group）内的模块按顺序执行，后面的模块可以覆盖或累加前面的值。下面详细展示每个组中的常用模块。

🟢 Emitter Spawn — 发射器创建时执行一次
在这个组里设置 Emitter 级别的初始值。通常不需要修改太多。
模块作用关键参数
Emitter Properties设置发射器基础属性Local Space, Sim Target (CPU/GPU), Fixed Bounds, Determinism
Spawn Burst Instantaneous瞬间生成一批粒子Spawn Count — 一次性生成数量
Emitter Life Cycle控制发射器的生命周期Loop Behavior, Loop Duration, Loop Delay
🔄 Emitter Update — 每帧执行（Emitter 级别）
控制粒子的持续生成速率和发射器的运行时行为。
模块作用关键参数
Spawn Rate按速率持续生成粒子SpawnRate — 每秒生成数量（如 20 = 每秒20个）
Spawn Burst Instantaneous瞬间爆发生成Spawn Count, Spawn Time, Spawn Probability
Spawn Per Frame每帧固定生成Spawn Count Per Frame
Emitter State管理发射器状态机Life Cycle Mode, Scalability Mode
✨ Particle Spawn — 每个粒子出生时执行一次
这是使用最频繁的组。在这里设置每个粒子的初始状态——位置、速度、颜色、大小、生命周期等。
模块作用关键参数
Initialize Particle粒子属性初始化（最重要！）Lifetime, Color, Sprite Size, Mass, Mesh Scale
Add Velocity添加初始速度Velocity (X,Y,Z) — 世界空间或局部空间
Shape Location在几何形状内生成粒子Shape Type (Box/Sphere/Cylinder/Cone/Torus/Mesh)
Set Particles.Position直接设置出生位置Position 向量
Set Particles.Color设置初始颜色RGBA 颜色值
Set Particles.SpriteSize设置精灵面片大小X/Y 大小（像素/单位）
Set Particles.SubImageIndex序列帧动画的初始帧Index 值
Camera Offset相对摄像机偏移Offset 距离
⚡
Initialize Particle 是最核心的模块，几乎每个 Emitter 都会用到。它集中初始化 Lifetime、Color、Size 等基础属性。右键可以展开查看所有可设置的子属性。
🔄 Particle Update — 每帧对每个存活粒子执行
让粒子在生命周期内动起来——颜色渐变、大小缩放、受力运动、碰撞检测等。
模块作用关键参数
Gravity Force施加重力Gravity — 默认 (0,0,-980)
Drag空气阻力Drag — 值越大减速越快 (0~∞)
Curl Noise Force柏林噪声力场（烟雾翻涌效果）Noise Strength, Noise Frequency, Pan Speed
Scale Color随生命周期缩放颜色/透明度Scale RGB, Scale Alpha — 支持曲线
Scale Sprite Size随生命周期缩放大小Scale Factor — 支持曲线
Solve Forces and Velocity力学求解器（将力转化为位移）必须放在所有力模块之后！
Collision粒子碰撞GPU: Scene Depth / Distance Field; CPU: Raycast
Sub UV Animation序列帧动画播放SubUV Lookup Mode, Sub Image Size
Sprite Facing and Alignment精灵面片朝向控制Face Camera, Custom Facing, Velocity Aligned
⚠️
Solve Forces and Velocity 必须放在所有 Force 模块（Gravity, Drag, Curl Noise 等）之后。它是力学求解器，负责将累加的力转化为速度和位置。如果顺序错了，力不会生效！
📡 Event Handler — 跨 Emitter 通信
让不同 Emitter 之间传递数据。例如：粒子碰撞时在碰撞点触发第二个 Emitter 产生火花。
机制说明典型用途
Generate Event发送端：在满足条件时发送事件数据碰撞时发送碰撞点位置和法线
Receive Event接收端：监听事件并生成新粒子在接收到的位置生成火花粒子
Death Event粒子死亡时自动发送事件粒子消失时在原地生成残留效果
Collision Event粒子碰撞时自动发送事件子弹击中地面时产生弹痕和尘土

Spawn 生成参数详解

生成参数决定了「粒子从哪里来、有多少、以什么初始状态出现」。这是效果设计的起点。

速度

VelocityVector3 | (X, Y, Z) 速度分量

Coordinate SpaceWorld / Local / Simulation
Velocity in Cone锥形范围内随机方向
生成量
生成模式对比
三种主要的粒子生成方式

Update 更新参数详解

Update 阶段决定粒子在存活期间的动态行为——运动轨迹、颜色变化、大小缩放等。这里是「让粒子活起来」的关键。

🎨 颜色与透明度控制
▶
模块作用参数
Scale Color随 NormalizedAge 缩放颜色/透明度Scale RGB (Curve), Scale Alpha (Curve) — 最常用于淡入淡出
Color by Speed根据粒子速度改变颜色Min Speed, Max Speed, Color Range
Set Color直接覆盖颜色可绑定 Dynamic Input 实现复杂逻辑
🎯
曲线编辑器技巧：点击 Scale Alpha 旁的下拉箭头 → 选择 Curve 类型 → 可以画出 0→1→0 的曲线实现「淡入 → 维持 → 淡出」效果。横轴是 NormalizedAge (0~1)，纵轴是缩放倍率。
📐 大小与缩放控制
▶
模块作用参数
Scale Sprite Size按曲线缩放 Sprite 大小Scale Factor (Curve/Float)
Scale Mesh Size按曲线缩放 Mesh 模型Scale Factor (Vector3)
Scale Sprite Size by Speed速度越快粒子越大/越小Min Speed, Max Speed, Scale Range

▶
参数说明值域
Sprite Rotation Rate每秒旋转角度float (度/秒)，可随机范围

Renderer 渲染器详解

渲染器决定粒子在屏幕上的可视化方式。选错渲染器 = 效果完全不对。下面是各类渲染器的对比和适用场景。

🔲 Sprite Renderer
最常用。将每个粒子渲染为一个面向摄像机的 2D 面片。

适用：火花、灰尘、烟雾、光点、UI 粒子
Material必须是 Unlit + Translucent 材质
SubImage Size序列帧动画的行×列
Sort ModeView Depth / Age / Custom
AlignmentUnaligned / Velocity / Custom
🧊 Mesh Renderer
将每个粒子渲染为 3D 模型实例。

适用：碎片、岩石、落叶、弹壳、冰块
Particle MeshStaticMesh 引用
Override Material可覆盖模型材质
Facing Mode默认使用粒子的旋转
Enable Frustum Culling视锥裁剪开关
🎗️ Ribbon Renderer
将同一 Emitter 的粒子连成一条带状轨迹。

适用：刀光拖尾、闪电弧线、轨迹线、彩带
Ribbon Width带状宽度
Max Tessellation细分精度
Facing ModeScreen / Custom / Custom Side
UV BehaviorStretch / Tile Based On Distance
💡 Light Renderer
每个粒子生成一个动态点光源。

适用：火焰照明、魔法光球、萤火虫
Radius Scale光源半径倍率
Intensity光源强度
Color Binding绑定粒子颜色
Affects Translucency是否照亮半透明物体
💡
渲染器选择速查：需要 2D 面片？→ Sprite | 需要 3D 模型？→ Mesh | 需要拖尾/轨迹？→ Ribbon | 需要粒子发光照亮场景？→ Light | 需要贴花？→ Decal
🌊
力与物理

力模块让粒子拥有物理行为。所有力模块必须在 Particle Update 中，且 Solve Forces and Velocity 必须放在最后。

⬇️ Gravity Force
施加恒定重力加速度
GravityVector3 | 默认 (0,0,-980)
UE 单位是厘米。-980 = -9.8 m/s²
🌬️ Drag
空气阻力，让粒子逐渐减速
Dragfloat | 0 = 无阻力，10+ = 快速停止
公式：速度每帧 × (1 - Drag × DeltaTime)
🌀 Curl Noise Force
柏林噪声力场，模拟烟雾/灵魂游荡的随机翻涌
Noise Strength力的强度 (越大越剧烈)
Noise Frequency噪声频率 (越高越碎)
Pan Speed噪声场移动速度
🎯 Point Force
从一个点向外推/向内吸引粒子
Force Strength正 = 推开，负 = 吸引
Force Position力场中心点世界坐标
Falloff衰减曲线
🌪️ Vortex Force
漩涡力，让粒子绕轴旋转
Vortex Axis旋转轴方向
Vortex Strength旋转力度
📌 Solve Forces & Velocity
必须的求解器！将所有力转化为实际位移。放在 Update 组最后。
位置必须在所有 Force 模块之后
Mass Clamp质量钳制范围
🧩
动态输入 Dynamic Inputs

Dynamic Input 是 Niagara 的灵魂功能之一——它让你不用写代码就能构建复杂的参数逻辑。点击任何参数值旁的下拉箭头，就能选择动态输入类型。

📈 Float from Curve
将一个浮点值绑定到曲线。横轴通常是 NormalizedAge，实现随时间变化的效果。
Curve IndexNormalizedAge / Custom
用途淡入淡出、大小缩放、速度衰减
🎲 Uniform Random
在范围内均匀随机取值。每个粒子获得不同的值，增加自然感。
Min / Max随机范围
Seed ModeDeterministic / Non-deterministic
🖩 Math Expression
直接写数学表达式。支持 +, -, *, /, sin, cos, lerp, clamp 等。
输入可引用任何 Particle/Emitter 参数
示例Particles.Velocity.Length() * 0.01
🗺️ Sample Texture
从贴图采样颜色/数据。可用于粒子沿贴图着色、读取 FlowMap 等。
TextureTexture2D 引用
UV采样坐标 (可绑定粒子位置)
🔀 Select / If
条件分支——根据布尔值或整数选择不同的输出。相当于代码中的 if/switch。
SelectorBool / Int / Enum
True/False Value不同条件下的输出值
📌 Make New / Map Get
构造新的结构体或从命名空间中读取参数。是参数间互相引用的桥梁。
Map Get读取 Particles.XXX, Emitter.XXX 等
Make Vector从分量构造向量
🔌
数据接口 Data Interface

Data Interface 让 Niagara 可以读取外部数据——骨骼、网格、贴图、曲线、音频、蓝图变量等。它是 Niagara 与引擎其他系统沟通的桥梁。

🦴 Skeletal Mesh DI
从骨骼网格表面/骨骼位置采样。

粒子沿角色身体表面生成

📐 Static Mesh DI
从静态网格表面采样位置/法线。

粒子沿模型表面分布

🖼️ Texture DI
采样 2D 贴图颜色值。

用贴图数据驱动粒子颜色/位移

📊 Curve DI
读取外部曲线资产做插值。

共享动画曲线给多个参数

🔉 Audio Spectrum DI
读取音频频谱数据。

音乐可视化效果

🧭 Spline DI
沿 Spline 组件采样位置/方向。

粒子沿路径运动

元素配方 — 对应「元素神殿」教程

以下是火焰、闪电、冰霜三大元素的 Niagara 参数配方思路，帮助你在跟随教程时快速理解每个参数的作用和调节方向。

🔥 火焰效果 Fire
火焰 = 向上喷发的粒子 + 颜色从亮黄到深红的渐变 + Curl Noise 翻涌 + 光照
1
Emitter — 生成配置
使用 Spawn Rate 持续生成（30~100/秒）。Shape Location 设为 Cylinder（底部圆形区域），使粒子从火堆底部涌出。
Spawn Rate: 50 | Shape: Cylinder (R=20, H=5) | Surface Only: false
2
Particle Spawn — 初始属性
Lifetime 设 0.5~1.5 秒（火焰粒子短命）。Add Velocity 向上 (Z = 200~400)。初始颜色设为明亮的黄白色 (HDR, 强度>1 以发光)。
Lifetime: Random(0.5, 1.5) | Velocity Z: Random(200, 400) | Color: (1.5, 1.2, 0.3, 1) HDR
3
Particle Update — 动态行为
Curl Noise Force 给火焰翻涌感（Strength=300, Freq=1.5）。Scale Color 曲线让颜色从黄→橙→红→透明渐变。Scale Sprite Size 先膨胀后缩小。Drag = 2 让上升减速。
Curl Noise: Strength=300, Freq=1.5 | Drag: 2 | Scale Alpha: 1→1→0 curve
4
Renderer + 材质
Sprite Renderer + 柔和噪声贴图材质（Unlit + Additive）。额外加一个 Light Renderer 让火焰照亮周围环境。材质中用 Particle Color 节点接入 Emissive。
Blend Mode: Additive | Emissive = ParticleColor * Intensity | Light Renderer: Radius=200, Intensity=5
⚡ 闪电效果 Lightning
闪电 = Beam/Ribbon 连线 + 随机偏移的锯齿路径 + 强烈的发光 + 短促的闪烁
1
Emitter — Beam 模式
使用 Beam Emitter 模板或手动设置。Spawn Burst 一次生成 10~20 个点（构成闪电段数）。设置 Beam Start/End 位置。
Spawn Burst: 15 points | Beam Start: Emitter Position | Beam End: Target Position
2
Particle Spawn — 路径偏移
每个 Beam 点在起点与终点之间插值，加入 Random Position Offset（横向偏移 ±30~80 单位）模拟闪电的锯齿感。Lifetime 极短（0.05~0.15 秒）。
Position: Lerp(Start, End, Index/Count) + Random Offset | Lifetime: Random(0.05, 0.15)
3
Renderer — Ribbon
使用 Ribbon Renderer，按粒子 ID 顺序连线。宽度中间粗、两端细。材质用高 HDR 亮度的蓝白色 + Fresnel 边缘发光。
Ribbon Width: Curve(thin→thick→thin) | Color: (3, 4, 8, 1) HDR Blue-White
4
进阶 — 分叉与闪烁
用 Event Handler 从主闪电上随机触发子 Emitter（分叉）。整体加 Emitter Update 中的随机 Alpha 闪烁（每隔几帧随机隐藏/显示）。
Branch Probability: 0.3 per segment | Flicker: Random Alpha 0~1 per frame
❄️ 冰霜效果 Ice / Frost
冰霜 = 冰晶碎片（Mesh）+ 地面扩散冻结（Decal/Sprite）+ 冷气雾气 + 闪烁光泽
1
Emitter A — 冰晶碎片
Mesh Renderer 使用低面数的冰晶模型。从中心向外爆发生成。Spawn Burst 一次性 10~30 个。Mesh 材质用半透明 + 折射 + 浅蓝色。
Mesh: IceShard_SM | Burst: 20 | Scale: Random(0.5, 2.0) | Color: (0.7, 0.9, 1.0)
2
Emitter B — 冰雾/冷气
低速、大粒子的 Sprite。Curl Noise 轻微翻涌（Strength=50）。颜色浅蓝半透明。Drag 高 (5~8) 让雾气缓慢漂浮。
Spawn Rate: 15 | Size: Random(80, 200) | Drag: 6 | Alpha: 0.15~0.3
3
Emitter C — 地面冻结扩散
使用 Decal Renderer 或平面 Sprite 贴在地面。从中心向外扩展（Scale 从 0 到 1 的曲线）。材质使用冰裂纹贴图 + 法线。
Scale Curve: 0→1 over 0.5s | Decal Size: 500 | Lifetime: 3~5s then fade

Aligned 模式（默认）= 左右手柄共线对接，拖动一端另一端自动镜像跟随，曲线过该点平滑连续——对应 UE 中的 Auto/Smooth 切线。
Broken 模式 = 左右手柄独立，可以制造尖角/急转——对应 UE 中的 Broken 切线。
双击控制点可在两种模式间切换，控制点中央的符号会变化（圆点 = Aligned，✕ = Broken）。

速查问答 — 常见问题
粒子不动？
检查 Particle Update 中是否有 Solve Forces and Velocity 模块，且在所有 Force 之后。
粒子看不见？
1. 检查 Renderer 是否存在 2. 材质是否正确绑定 3. Sprite Size 是否太小 4. Color Alpha 是否 > 0
CPU 还是 GPU 模拟？
Emitter Properties → Sim Target。GPU 支持百万粒子但无法精确碰撞；CPU 精确但数量受限（千级）。
Local Space 和 World Space 区别？
Local Space: 粒子跟随 System 移动。World Space: 粒子生成后留在原地。火焰用 Local，拖尾用 World。
怎么让粒子颜色随时间变？
Particle Update → Scale Color。将 Scale RGB 和 Scale Alpha 设为 Float from Curve，横轴绑定 NormalizedAge。
Ribbon 拖尾断裂？
检查 Ribbon ID 绑定是否正确。粒子必须有连续的 ID。也可能是 Spawn Rate 太低导致点间距太大。
怎么让不同 Emitter 交互？
使用 Event Handler。发送端 Generate Event → 接收端 Receive Event。可传递 Position, Velocity, Color 等。
Niagara 性能优化？
1. 降低 Spawn Rate 2. 缩短 Lifetime 3. 用 GPU Sim 4. 设 Fixed Bounds 5. 减少 Overdraw（调小 Sprite Size）6. Scalability 降级
怎么用蓝图控制 Niagara？
在 System 中定义 User Parameters，蓝图中用 Set Niagara Variable 节点运行时修改。支持 Float, Vector, Color 等类型。
Custom Module 怎么写？
在 Scratch Pad 中创建模块。用节点图或 Custom HLSL 表达式。输入/输出绑定到 Map Get / Map Set。
粒子穿模了？
添加 Collision 模块。GPU 用 Scene Depth 或 Distance Field；CPU 用 Raycast。记住碰撞模块要在 Solve 之前。
Deterministic 是什么？
确定性模拟 = 相同种子和参数下每次播放效果完全一致。适用于过场动画。Emitter Properties → Determinism 开启。

◆ 效果设计思考框架 — 「五问法」

每次设计一个新的 VFX 效果时，按以下 5 个维度逐一思考，能帮助你快速拆解需求、避免遗漏。

Q1 形态是什么？
决定视觉呈现方式，直接关联 Renderer 选型。
  1. 点状/片状 → Sprite Renderer
  2. 3D 碎片/模型 → Mesh Renderer
  3. 线条/拖尾/弧线 → Ribbon Renderer
  4. 地面痕迹 → Decal Renderer
  5. 自发光照亮场景 → 额外加 Light Renderer
Q2 从哪里来？
决定粒子的 生成方式和初始位置。
  1. 一个点 → 默认 / Set Position
  2. 几何形状区域 → Shape Location
  3. 沿模型表面 → Mesh / Skeletal Mesh DI
  4. 沿路径生成 → Spline DI
  5. 爆发 vs 持续 → Burst vs Rate
Q3 怎么动？
决定运动行为，配置 力场和速度。
  1. 直线飞射 → Add Velocity + 低 Drag
  2. 受重力抛物线 → Gravity Force
  3. 随机翻涌 → Curl Noise Force
  4. 旋转涡流 → Vortex Force
  5. 被吸引/推开 → Point Force
Q4 怎么变？
决定生命周期内的 视觉渐变。
  1. 颜色渐变 → Scale Color + 曲线
  2. 大小缩放 → Scale Sprite Size + 曲线
  3. 透明度淡入淡出 → Scale Alpha 曲线
  4. 旋转 → Sprite Rotation Rate
  5. 序列帧动画 → SubUV Animation
Q5 怎么交互？
决定粒子与环境/其他系统的 互动方式。
  1. 碰撞地面 → Collision 模块
  2. 触发次级效果 → Event Handler
  3. 蓝图运行时控制 → User Parameters
  4. 响应音频 → Audio Spectrum DI
  5. LOD/Scalability → 性能分级
💡 调试检查清单
效果不对时的排查顺序：
  1. Renderer 存在？材质正确？
  2. Spawn Rate > 0？Lifetime > 0？
  3. Size/Color/Alpha 是否可见？
  4. Solve Forces 在力模块之后？
  5. Local/World Space 选对了？
  6. GPU vs CPU 模式匹配？
